# phase1-local-chat-plan

# Phase 1（本地优先，不含远端服务）实施计划与技术方案

## 目标
- 在 Expo 聊天应用内实现本地可靠消息能力（不依赖远端服务）。
- 覆盖三块核心能力：
  - SQLite 消息持久化
  - 会话索引（最后消息、未读数、更新时间）
  - 发送队列与失败重试

## 范围（Phase 1）
- 仅本地能力，不包含服务端 API、WebSocket、远端同步。
- 仅实现文本消息链路（附件能力保留扩展点）。
- 提供可复用的仓储与队列模块，供 `src/hooks/useChat.js` 后续接入。

## 技术方案
- 数据存储：`expo-sqlite`（异步 API）
- 本地模型：
  - `conversations`：会话索引与未读统计
  - `messages`：消息明细
  - `outbox_queue`：发送任务队列（pending/sending/failed/sent）
- 排序与索引：
  - `messages(conversation_id, created_at_client DESC)`
  - `messages(conversation_id, client_id UNIQUE)`
  - `outbox_queue(status, next_retry_at, created_at)`
- 发送策略：
  - 本地先落库（pending）并更新会话摘要
  - 队列后台拉取待发送任务
  - 成功：消息与队列状态写为 sent
  - 失败：指数退避更新 `next_retry_at`，状态改 failed（可继续重试）

## 交付文件
- `docs/Phase1/phase1-local-chat-storage.js`
  - 数据库初始化（建表、索引）
  - 会话/消息仓储接口
  - 发送队列入队、拉取、确认、失败回写
  - 重试调度器（可被 UI 定时调用）

## 实施步骤
- 定义常量与表结构 SQL
- 实现 DB 初始化与事务封装
- 实现消息/会话写入与查询接口
- 实现 outbox 队列与重试逻辑
- 提供接入示例（如何在 `useChat` 中替换内存消息）

## 验收标准
- 文本消息发送后可在本地持久化
- 重启应用后消息仍可读取
- 失败任务进入重试并可最终变更状态
- 会话列表能读取 last message + unread + updated_at
